# Алгоритмы CURSA: как использовать схемы

В каталоге находятся блок‑схемы основных процессов системы в формате Mermaid (`flowcharts.md`). Этот формат поддерживается GitHub и рядом расширений для VS Code. При необходимости схемы можно экспортировать в изображения для включения в отчёты.

Рекомендации по оформлению, близкие к ГОСТ 19.701‑90:
- Единый поток сверху вниз, один вход и один выход у каждой схемы.
- Решения — ромбы с подписями ветвей «ДА/НЕТ».
- Начало/Конец — скруглённые блоки; ввод/вывод — параллелограмм; процесс — прямоугольник.
- Наименования шагов короткие, по делу («Сохранить», «Проверить», «Удалить», «Отдать ответ»).

Экспорт (по желанию):
- Через web‑рендереры Mermaid (например, mermaid.live) — вставьте содержимое блока и скачайте PNG/SVG.
- Через Mermaid CLI (если установлен Node.js и `@mermaid-js/mermaid-cli`): сгенерируйте изображения из `.md` по блокам кода.

Поддержка профилей норм:
- Схема «Загрузка и применение профиля» отражает будущий модуль конфигураций: проверка профиля по JSON Schema, хранение в `backend/profiles/`, кэширование и применение в `NormControlChecker`.
